# AGENTS.md

『부의 게임, 대충 뛰어들어 끝까지 살아남기』 저장소에서 AI가 작업할 때 따라야 하는 운영 문서다.

이 저장소는 책 원고뿐 아니라 재무 강의, 재무 코칭, 블로그 포스팅, 콘텐츠 마케팅 관련 자료까지 함께 관리한다.

작업 전 반드시 먼저 확인할 문서:
- `README.md`
- `guides/WRITING_RULES.md`
- `guides/DISPLAY_GUIDE.md`
- `Additionals/RULES.md` (`Additionals/` 하위 작업 시 필수)

반드시 지킬 원칙:
- 문체는 기본적으로 '~이다', '~한다' 평어체를 사용한다.
- 게임 메타포를 유지하되 과장된 장식보다 이해를 돕는 방향으로 사용한다.
- 독자의 불안을 자극하는 어조보다 '따뜻하고 대충 합리적인' 태도를 우선한다.
- 일반 챕터는 `챕터 요약 -> 챕터 일러스트 -> 체크인 질문 -> 본문 -> 퀘스트 완료 레벨업 질문` 구조를 따른다.
- 부록 챕터는 질문 섹션 없이 설명형 구조로 작성한다.
- 마크다운 강조 문법은 렌더링이 깨지지 않도록 경계를 점검한다.
- 목차, 장 제목, 파일 구성이 바뀌면 `README.md` 링크도 함께 갱신한다.
- 책 원고는 `chapters/`에 저장하고, 책 외 콘텐츠는 반드시 `Additionals/` 아래에 저장한다.
- `Additionals/` 관련 상세 저장 구조와 콘텐츠 범위 규칙은 `Additionals/RULES.md`를 따른다.

작업 방식:
- 작업은 한 번에 크게 밀어붙이지 말고 단계별로 진행한다.
- 사용자가 이해하지 못했거나 동의하지 않은 내용은 프로젝트 내용으로 확정해서 넣지 않는다.
- 설명이 필요하거나 해석 여지가 큰 내용은 먼저 의미를 분명히 설명하고, 사용자의 이해를 확인한 뒤 반영한다.
- 새 원고를 쓰기 전 해당 챕터와 인접 챕터의 톤과 구조를 먼저 확인한다.
- 수정 작업은 문장만이 아니라 구조, 질문, 가독성 기준까지 함께 점검한다.
- `Additionals/` 자료를 만들 때는 먼저 연결되는 `chapters/` 근거를 확인하고 관련 챕터 링크를 포함한다.
- 상세 기준이 충돌하거나 애매하면 `guides/WRITING_RULES.md`와 `guides/DISPLAY_GUIDE.md`를 우선 기준으로 삼는다.
