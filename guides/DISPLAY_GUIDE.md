# 📱 Display & Layout Guide (가독성 가이드)

『부의 게임, 대충 뛰어들어 끝까지 살아남기』를 독자가 화면에서 읽을 때 최상의 사용자 경험(UX)을 제공하기 위한 서식 및 레이아웃 규칙입니다.

---

## 1. 헤더 구조 (Hierarchy)

- **# (H1)**: 챕터의 전체 제목. (파일당 1개만 사용)
- **챕터 요약**: 챕터 제목 직후에 나오는 한두 문단의 요약문으로, 독자에게 다정한 길안내를 제공한다.
- **챕터 일러스트**: 요약문 바로 아래, 가독성을 위해 중앙 정렬된 이미지로 배치한다.
- **## (H2)**: 파트 내 큰 섹션 구분.
- **### (H3)**: 섹션 내 세부 항목이나 소제목.

## 2. 텍스트 강조 및 스타일링

- **중요 문구 (중앙 정렬)**: 독자에게 강하게 전달해야 하는 핵심 문구는 다음과 같이 중앙 정렬과 볼드체로 표시하며, 전후에 반드시 빈 줄을 두어 독립된 영역으로 보이게 한다. 일부 브라우저 Markdown 렌더러는 HTML 블록 안의 `**` 강조를 처리하지 못할 수 있으므로, HTML 블록 안에서는 `<strong>`을 사용한다.
  ```markdown
  <div align="center">

  <strong>"여기에 강조할 핵심 문장을 적습니다."</strong>

  </div>
  ```
- **인용구**: 명언이나 외부 인용문은 이탤릭체(`*텍스트*`) 또는 인용 블록(`> 텍스트`)을 사용하여 본문과 구분한다.
- **볼드체**: 단어 강조 시 `**텍스트**`를 사용하되, 렌더링 결과에서 기호가 텍스트처럼 노출되지 않는지 확인한다.

## 3. 가독성을 위한 간격 (Spacing)

- **섹션 구분**: 각 섹션(##) 사이에는 최소 2줄 이상의 빈 줄을 두어 시각적으로 분명하게 나눈다.
- **체크인/레벨업 섹션**: 본문과 뚜렷하게 구분될 수 있도록 섹션의 시작과 끝에 가로 구분선(`---`)을 넣고, 각 질문은 인용 블록(`>`)을 사용하여 독립된 공간처럼 표현한다. 질문 간에는 반드시 빈 줄을 두어 가독성을 높인다.
- **리스트 (1, 2, 3)**: 목록형 데이터 작성 시 항목 간 줄바꿈을 명확히 하여 텍스트가 뭉쳐 보이지 않게 한다.
- **단락**: 하나의 단락이 너무 길어지지 않도록 적절히 나누어 모바일 환경에서도 읽기 편하게 한다.

## 4. 시각 자료 활용

- **이미지**: 본문 내용을 보완하는 이미지는 중앙 정렬로 배치하며, 이미지 아래에 짧은 캡션을 추가할 수 있다.
- **TIP/경고**: 특별히 주의해야 하거나 유용한 팁은 인용 블록이나 별도의 강조 기법(GitHub Alerts 등)을 사용한다.

---

## 5. 부록(Appendix) 챕터 전용 레이아웃

부록 챕터는 일반 챕터와 시각적으로 차별화된 구조를 갖는다.

- **용도 안내 (Introduction)**: 챕터의 제목 직후에 이 부록이 어떤 내용을 담고 있으며 어떻게 활용해야 하는지 명확히 밝히는 단락을 구성한다.
- **질문 섹션 제외**: 질문을 위한 가로 구분선(`---`)과 인용 블록(`>`)을 사용하지 않는다.
- **섹션별 구분**: 설명이 필요한 각 항목은 명확한 헤더(H2, H3)를 사용하여 독립적인 섹션으로 나누어 가독성을 높인다.
- **챕터 내 참조**: 부록과 직접 이어지는 표, 용어, 계산 가정이 있을 때만 본문의 흐름을 방해하지 않는 위치에 짧게 안내한다. 모든 챕터에 같은 부록 안내 문장을 반복하지 않는다.

---

## 5. 뷰어 확인 사항

- 작성 후에는 반드시 Markdown 뷰어로 확인하여 `**`, `_` 등의 문법 기호가 텍스트처럼 그대로 노출되는 현상이 없는지 체크한다.
- `WRITING_RULES.md`는 집필 원칙, `DISPLAY_GUIDE.md`는 화면 표시와 가독성 점검 기준으로 역할을 나눈다.
- 모든 서식은 글의 흐름을 방해하지 않고 독자가 자연스럽게 읽어 내려갈 수 있도록 적용한다.
