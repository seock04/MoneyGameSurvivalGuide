# Completion Rubric

총점은 100점이다. 각 영역은 원고 전체 기준으로 평가한다.

## Weighted Score

| 영역 | 배점 | 평가 기준 |
|---|---:|---|
| 1. 핵심 메시지와 책 구조 | 15 | 목차 흐름, 챕터 순서, 반복/누락, 프롤로그-본문-부록-에필로그의 역할 분담 |
| 2. 독자 여정과 이해 가능성 | 15 | 초보 독자가 문맥 점프 없이 따라오는지, 용어 정의와 예시가 충분한지 |
| 3. 재무 정확성과 투자 안전성 | 15 | 사실 정확성, 수익률/통계 해석, 보장형 표현 회피, 추천/교육 경계 |
| 4. 톤과 심리적 안전성 | 12 | 따뜻하고 대충 합리적인 태도, 불안/죄책감 자극 최소화, 자기효능감 |
| 5. 실행 가능성 | 10 | 독자가 바로 할 수 있는 작은 행동, 질문 품질, 자동화/습관 설계 |
| 6. 출처와 데이터 무결성 | 10 | Sources 존재, 최신성, 출처-본문 수치 일치, 직접 인용/저작권 안전성 |
| 7. 표시 품질과 마크다운 렌더링 | 8 | `DISPLAY_GUIDE.md` 준수, `**` 노출 위험, 표/이미지/간격/HTML 블록 |
| 8. 시각 자료 완성도 | 5 | 이미지 품질, 장면 적합성, 스타일 일관성, 파일 링크 정상 |
| 9. 파일/링크/메타 정합성 | 5 | README 목차, 파일명, 챕터 번호, 내부 참조, status 기록 |
| 10. 출간 전 polish | 5 | 오탈자, 문장 리듬, 중복 표현, 최종 독서감 |

## Score Bands

| 점수 | 판정 | 의미 |
|---:|---|---|
| 99.0~100 | READY_FOR_FINAL_HUMAN_REVIEW | 출간 전 사람 최종 검토 단계 |
| 95.0~98.9 | NEAR_READY | 큰 구조는 안정적이나 일부 고영향 polish 필요 |
| 90.0~94.9 | REVISION_REQUIRED | 독자 경험 또는 신뢰도에 영향을 주는 문제가 남음 |
| 80.0~89.9 | MAJOR_REVISION | 장별 구조/팩트/톤 중 하나 이상이 흔들림 |
| ~79.9 | REBUILD_NEEDED | 전체 설계 재검토 필요 |

## Hard Gates

아래 항목 중 하나라도 실패하면 총점이 높아도 99% 판정 불가다.

- 투자 수익, 내집마련, 경제적 자유를 보장하는 문장이 없다.
- 특정 종목/상품 매수를 직접 권유하는 문장이 없다.
- 최신 수치, 법/제도, 가격, 수익률에는 확인 가능한 출처가 있다.
- 모든 일반 챕터에 요약, 일러스트, 체크인 질문, 본문, 레벨업 질문이 있다.
- 부록 챕터는 질문 섹션 없이 설명형 구조를 유지한다.
- 브라우저 렌더링에서 `**`, `_`, 깨진 이미지 링크가 그대로 보이는 문제를 해결했다.
- README 목차와 실제 파일 구성이 일치한다.
- Critical 팩트체크 이슈가 없다.

## Severity

| 등급 | 기준 | 처리 |
|---|---|---|
| Critical | 법적/재무적 오해, 명백한 사실 오류, 링크/구조 붕괴, 독자에게 해로운 단정 | 즉시 수정, 99% 판정 불가 |
| High | 독자 이탈 가능성이 큰 설명 부족, 구조 중복, 강한 톤 문제, 핵심 데이터 출처 부족 | 다음 라운드 우선 수정 |
| Medium | 가독성, 예시 보강, 문장 리듬, 표 정리 | 라운드 여유에 따라 수정 |
| Low | 취향성 polish, 단어 선택, 미세한 반복 | 마지막 polish 단계에서 처리 |

## Scoring Rules

- 점수는 감점식으로 계산한다.
- 동일 문제가 여러 장에 반복되면 반복 패턴으로 묶어 한 번에 감점하되, 영향 범위를 적는다.
- 새 내용을 추가해 문제를 해결할 때는 분량 증가로 인한 페이싱 저하도 함께 평가한다.
- 99점 이상에서는 Low 이슈도 독자 경험에 실제 영향을 주는지 따져 과잉 수정하지 않는다.
