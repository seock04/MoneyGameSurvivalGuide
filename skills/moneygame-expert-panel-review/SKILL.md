# MoneyGame Expert Panel Review

## Overview

이 스킬은 가상 6인 페르소나 패널이 책 원고(`chapters/*.md`)를 리뷰한 것처럼, 문제를 발견하고 "수정 방향"을 제안한다.

목표:
- 일반 독자가 경제 관념과 심리를 "제대로 이해"하도록 돕는다.
- 문맥 점프/어려운 용어/설명 부족으로 인한 이탈을 줄인다.
- 한국어 문장 구성, 문법, 조사, 호응, 문맥 연결이 어색해 생기는 이탈을 줄인다.
- 책의 철학(따뜻하고 대충 합리적, 게임 메타포, 생존 우선, 자동 사냥 시스템)을 더 단단하게 만든다.

산출물:
- 리뷰 리포트: `status/expert_panel_review/YYYY-MM-DD.md`

핵심 제약:
- 단계별로 진행한다. 사용자가 이해/동의하지 않은 내용은 원고에 확정 반영하지 않는다. (`AGENTS.md`)
- 기본 모드는 "제안"이다. 사용자 승인 없이 `chapters/`를 직접 수정하지 않는다.
- 책 밖의 정보는 원칙적으로 보조 수단이며, 추가가 필요하면 근거와 목적을 함께 제시한다. (`Additionals/RULES.md`의 취지 준수)
- 기본은 "일괄 리뷰"가 아니라 "단계별(소량) 리뷰"다: 1회 실행에서 1개 챕터(또는 사용자가 지정한 소수 챕터)만 리뷰한다.
- 00~14 전체 일괄 리뷰는 사용자가 명시적으로 요청할 때만 수행한다.

## Review Lenses (6 personas)

페르소나 세부 규칙은 `references/panel_personas.md`를 따른다.

1. Nobel Literature Lens
   - 문장 리듬, 이미지, 비유의 정확성, 장면 전환, 독자 몰입
   - 챕터 단독 서사: 한 챕터만 읽어도 "시작-전환-끝"이 있고, 전달 메시지가 분명히 드러나는지
   - 주목도(훅): 초반 10~20% 구간에서 독자가 계속 읽을 이유가 생기는지
   - 질문 프레이밍: 체크인/레벨업 질문이 "챕터의 메시지"를 감싸며 사고를 자극하는지(형식만 있고 내용이 약하지 않은지)
2. World-class Finance Expert Lens
   - 사실/정의 정확성, 과장/단정 방지, 오해 여지, 안전한 표현
3. World-class Financial Coach Lens
   - 실행 가능성, 과제 설계, 단계화, 독자 불안 완화, 행동 변화 유도
4. World-class Psychologist Lens
   - 인지 왜곡/불안 트리거, 자기효능감, 죄책감 유발 문장, 동기/습관 설계
5. Stand-up Behavioral Econ Communicator Lens
   - 위트/유머로 이해를 돕는지, 과장 없이 단순화하는지, 공포 마케팅을 피하면서도 행동을 촉발하는지
6. Korean Language Scholar Lens
   - 한국어 문장 구성, 조사/어미/호응, 문맥 연결, 어색한 번역투, 의미 충돌 표현을 점검하는지

## What To Flag

- 문맥 점프: 앞 문단의 전제가 없이 결론/용어/사례로 이동
- 어려운 용어: 정의 없이 등장, 괄호 정의가 불충분, 예시/비유 없이 등장
- 과도한 확신/공포: 독자의 불안을 키우는 단정형
- 지시 과잉: 독자가 "해야 한다" 압박만 받고 어떻게 시작할지 모르는 상태
- 실행 설계 미흡: 다음 행동이 너무 크거나 추상적
- 메타포 과잉/혼재: 게임 비유가 이해를 돕기보다 혼란을 만드는 경우
- 메시지 불명확: 이 챕터에서 독자가 "그래서 뭐가 핵심이지?"가 남는 경우(한 줄로 요약이 안 되는 경우)
- 챕터 서사 부족: 장면/전환/결론이 약해 챕터 단독으로 읽었을 때 힘이 빠지는 경우
- 질문 품질: 열린 질문이 아닌 단답형, 질문이 너무 많거나(한 불렛에 복수 질문), 독자가 자기 상태를 돌아보고 답하기 어려운 질문
- 유머 톤: 농담이 개념을 흐리거나 독자를 비웃는 느낌이 되지 않는지, 반대로 너무 건조해 이탈을 유발하지는 않는지
- 구조 문제: 한 챕터에 주제가 과밀해 분리/추가/재배치가 필요하거나, 목차/파일명이 내용을 제대로 반영하지 못하는 경우
- 반복/중복: 같은 설명/비유/문장 구조가 무분별하게 반복되어 독자가 피로해지거나 "이미 읽었다" 이탈이 생기는 경우
- 분량/페이싱: 챕터 길이가 과밀하거나 호흡(장문/반복/우회)이 길어 핵심 전달이 늦어지는 경우
- 한국어 문장 어색함: 조사/어미/주술 호응이 맞지 않거나, 번역투·중복 수식·의미 충돌 표현 때문에 문장이 자연스럽지 않은 경우

## Review Flow (fixed order)

리뷰는 아래 순서로만 진행한다. (순서를 섞지 않는다)

1. Scope: 범위 확정(챕터/섹션)
2. Structure: 구조/가독성 빠른 스캔(이탈 지점 후보 표시)
3. Questions: 체크인/레벨업 질문 품질 점검
4. Panel: 6인 페르소나 패널 리뷰(장 단위)
5. Proposals: 수정 방향 제안(브리지/용어 풀기/문장 다듬기/실행 쪼개기)
6. Report: status 리포트 저장 + 다음 액션 1개

## Structural Change Protocol (when needed)

리뷰 중 "챕터 분리/추가/재배치/제목 변경" 같은 구조적 변경이 필요하다고 판단되면, 아래 규칙으로만 대응한다.

- 기본 원칙: 사용자 승인 전에는 구조를 바꾸지 않는다(제안만 한다).
- 제안은 작은 단위로 쪼갠다: 한 번에 하나의 구조 변경만 제안한다.
- 모든 제안에는 근거(독자 관점 문제)와 기대 효과(이해/몰입/실행)가 포함되어야 한다.

절차:
1. Trigger 기록: 무엇이 구조 변경을 요구하는가(문맥 점프, 용어 과밀, 주제 혼재, 반복 등)
2. Options 제시: 최소 2안
   - A안: 현 챕터 유지 + 문장/섹션 보강으로 해결
   - B안: 챕터 분리/추가/재배치(파일명/목차 변화 포함)
3. Diff 형태로 명시: 변경될 파일/목차/섹션을 구체적으로 적는다
4. 리스크 표시: 링크 깨짐(README), 톤/중복, 독자 흐름, 작업량
5. 승인 후 실행: 사용자가 선택한 1안만 수행(그때 README 링크도 함께 갱신)

## Workflow (Step-by-step)

1. 범위 확정
   - 기본: 1개 챕터만 선택(예: `chapters/01_...md`)
   - 사용자가 요청한 경우에만 여러 챕터를 선택
   - 00~14 전체는 "명시 요청"이 없으면 선택하지 않는다
   - 기본 선택 규칙(사용자가 지정하지 않았을 때):
     - 아직 리뷰하지 않은 가장 앞 챕터 1개를 선택한다(00 -> 14 순서).
     - 13장(부록)은 질문 섹션 예외이므로 Questions 단계는 "해당 없음"으로 표시한다.

2. 빠른 스캔(구조/가독성)
   - 섹션 구조, 질문 섹션, 일러스트 자리표시
   - 장별 "독자 이탈 지점" 후보 표시
   - 반복/분량 신호를 함께 표시한다:
     - 같은 설명이 다른 섹션에서 재등장하는지
     - 핵심 메시지까지 도달하는 데 너무 오래 걸리는지
   - Nobel 관점에서 "훅/전환/결말"이 보이는지도 함께 표시한다:
     - 초반에 독자를 붙잡는 1문장(또는 장면)이 있는지
     - 중간에 메시지를 한 번 더 선명하게 만드는 전환(정의/대조/사례)이 있는지
     - 끝에 가서 "그래서 나는 무엇을 바꾸면 되지?"가 남는지

3. 질문(체크인/레벨업) 품질 점검
   - 질문은 열린 질문인지(YES/NO, 단답 회피) 확인한다.
   - 한 불렛에 질문이 1개만 있는지 확인한다.
   - 독자가 "현재 상태를 돌아보는 답"을 쓰기 쉬운지(구체성/부담/선명도)를 점검한다.
   - 필요한 경우: 질문을 더 쉽게 풀어 쓰는 수정 방향만 제안한다(원고 직접 수정 금지).

4. 패널 리뷰(장 단위)
   - 각 장에서 6 렌즈로 최소 1개씩: "좋은 점 1, 개선점 1"을 기록
   - 개선점은 "왜 문제인지"와 "어떻게 고칠지"를 분리한다

5. 제안 설계(보수적으로)
   - 용어는 "한 줄 정의 -> 예시 1개 -> 오해 1개"로 풀어쓴다
   - 문맥 점프는 "브리지 문장 1개"로 연결한다
   - 실행 과제는 "10분짜리 1개"로 쪼갠다

6. 리포트 저장
   - `status/expert_panel_review/YYYY-MM-DD.md`에 요약/장별 제안/다음 액션을 기록한다
   - 리포트는 반드시 `status/expert_panel_review/template.md`의 필수 필드를 채운다.
   - 기본은 "1회 실행 = 1챕터"이므로, 리포트의 `범위`에는 해당 챕터 파일 경로 1개만 넣는다.
