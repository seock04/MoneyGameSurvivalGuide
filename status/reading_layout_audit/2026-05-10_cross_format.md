# Reading Layout Audit: Cross-Format Review

Date: 2026-05-10  
Scope: `chapters/*.md`  
Rule set: `moneygame-reading-layout-auditor` updated Content Shape Decision, checklist, and cross-format rules

## Verdict

**CROSS_FORMAT_REVISIONS_RECOMMENDED**

마크다운 원고 자체는 치명적인 렌더링 오류 없이 읽을 수 있다. 자동 스캔 기준으로도 Critical/High는 없고, Medium은 `00_목차.md`의 긴 목차 목록 1건이다.

하지만 강화된 기준으로 보면 핵심은 다르다. 책 원고는 종이책, 모바일 ebook, 태블릿, PC, Word, 블로그, PPT로 옮겨질 수 있는 원천 콘텐츠다. 따라서 단순히 내용이 맞는지만 볼 것이 아니라, 독자가 한 화면 안에서 **핵심 메시지, 근거, 실행할 행동**을 구분할 수 있어야 한다.

현재 원고는 전체적으로 따뜻한 설명과 게임 메타포는 잘 유지하고 있다. 다만 데이터 표와 비교 설명이 많은 구간에서 “좋은 내용이 한 덩어리로 붙어 보여 전달력이 약해지는 문제”가 남아 있다.

## Priority Findings

### 1. `chapters/01_부의_게임_시작하기.md`: 5열 생활감 비교표가 모바일 ebook에서 무겁다

Location: `chapters/01_부의_게임_시작하기.md` around line 79

Current shape:

- `품목/행동`
- `1985년 가격대 예시`
- `1985년 10,000원으로 가능한 양`
- `2025년 전후 가격대 예시`
- `2025년 10,000원으로 가능한 양`

Problem:

이 표는 정보 자체는 좋지만, 5열 구조라 모바일 ebook, Word 좁은 페이지, PPT에서 가독성이 크게 떨어진다. 독자가 읽어야 하는 메시지는 “1985년의 10,000원은 지금보다 훨씬 묵직했다”인데, 현재는 숫자 비교가 먼저 보이고 핵심 메시지가 뒤따라온다.

Required change:

- 표를 2~4개의 `품목별 비교 블록`으로 바꾸거나, `1985년 / 2025년 / 독자가 느낄 차이` 3열 이하로 줄인다.
- 각 품목은 한 줄 비교보다 아래 구조가 낫다.
  - 핵심: `라면: 약 100봉에서 약 10~12봉으로 줄어든 감각`
  - 근거: `1985년 약 100원, 2025년 약 800~1,000원`
  - 의미: `같은 만 원이라도 생활 선택권이 줄었다`
- PPT 전환 시에는 표가 아니라 `라면, 자장면, 버스, 우유` 4개 타일 또는 한 장의 비교 그래픽으로 만든다.

### 2. `chapters/14_부록_부의_게임_인벤토리_상세_가이드.md`: 데이터 표 묶음이 주장보다 앞선다

Location: `chapters/14_부록_부의_게임_인벤토리_상세_가이드.md` sections `5-1` through `5-4`, `8`

Problem:

14장은 부록이라 데이터가 많은 것은 자연스럽다. 다만 현재는 `[표]`, 표, `[그래프]`, 코드형 그래프, 설명이 반복되면서 독자가 “그래서 이 표를 어떻게 읽어야 하는가”를 표를 본 뒤에야 알게 되는 구간이 있다.

Required change:

- 각 표 앞에 한 문장짜리 주장형 리드문을 둔다.
  - 예: `이 표의 핵심은 환율 예측이 아니라, 원화 생활자에게 달러 가격표가 삶의 선택권에 영향을 준다는 점이다.`
- `[표]`, `[그래프]` 라벨은 Markdown 문단 라벨보다 `#### 표 제목` 또는 캡션 규칙으로 통일한다.
- `5-2`의 지수 변화, 상승 감각, 읽는 법은 한 덩어리로 붙이지 말고 다음 순서로 재배치한다.
  - 핵심 주장
  - 비교 표
  - 읽는 법
  - 주의점
  - 본문으로 돌아가는 의미
- PPT 전환 시에는 표 전체를 가져가지 말고 `핵심 숫자 3개 + 해석 1문장`으로 축약한다.

### 3. `chapters/14_부록_부의_게임_인벤토리_상세_가이드.md`: 장기 수익률 표 3개는 책에는 가능하지만 PPT/블로그에는 과밀하다

Location: `chapters/14_부록_부의_게임_인벤토리_상세_가이드.md` around lines 240, 255, 270

Problem:

1995~2025년 S&P 500 수익률을 3개 표로 나눈 것은 모바일 폭에는 유리하다. 그러나 독자가 실제로 기억해야 하는 메시지는 “매년 일정히 오르지 않는다. 큰 하락과 회복이 섞인다”이다. 현재는 연도별 숫자가 먼저 보이기 때문에 기억할 메시지가 늦게 도착한다.

Required change:

- 표 앞에 `읽을 포인트 3개`를 둔다.
  - 큰 하락은 주기적으로 온다.
  - 회복도 같은 표 안에 있다.
  - 그래서 투자 성과보다 유지 시스템이 먼저다.
- 책에서는 3개 표를 유지해도 되지만, 블로그/PPT용으로는 `최대 하락 연도`, `강한 회복 연도`, `전체 교훈` 요약 박스를 추가한다.
- PPT 전환 시에는 연도별 표 대신 막대 차트 또는 `하락-회복-유지` 3단 구조가 적합하다.

### 4. `chapters/00_목차.md`: 목차가 긴 단일 목록이라 책의 큰 여정이 약하게 보인다

Location: `chapters/00_목차.md` line 5

Problem:

17개 항목이 단일 목록으로 이어진다. 책의 흐름은 존재하지만 독자에게는 “챕터 나열”로 보일 수 있다. 특히 모바일 ebook 첫 화면에서는 전체 구조를 기억하기 어렵다.

Required change:

- 목차를 파트 단위로 묶는다.
  - 시작과 관점
  - 자동화와 현재 좌표
  - 투자와 성장
  - 위험 관리와 확장
  - 부록과 마무리
- PPT/블로그 전환 시 각 파트가 시리즈 카테고리나 섹션 헤더로 그대로 쓰일 수 있게 한다.

### 5. Global: 이미지와 캡션이 `<div align="center">`에 의존한다

Location: most chapter hero images

Problem:

현재 방식은 GitHub Markdown과 일부 Markdown 렌더러에서는 잘 보인다. 다만 HTML, Word, PPT 전환에서는 의미 있는 `figure`, `figcaption`, Word Caption, PPT image placeholder로 자동 매핑하기 어렵다.

Required change:

- 원고 Markdown에서는 기존 규칙을 유지해도 된다.
- 변환 규칙에는 다음 매핑을 명시해야 한다.
  - Markdown: 이미지 + 바로 아래 italic caption
  - HTML/blog: `<figure><img ...><figcaption>...</figcaption></figure>`
  - Word: 이미지 + `Caption` 스타일
  - PPT: 이미지 placeholder + 짧은 assertion title
- 실제 변환 작업 시 `<div align="center">`는 표시용 태그로만 보고 의미 구조는 캡션에서 복원한다.

### 6. Global: `[표]`, `[그래프]`, `[도표]` 라벨이 포맷별 의미 구조로 고정되지 않는다

Location: `chapters/01_부의_게임_시작하기.md`, `chapters/14_부록_부의_게임_인벤토리_상세_가이드.md`

Problem:

사람이 읽으면 제목처럼 보이지만, Word/PPT/HTML 변환에서는 캡션, 소제목, 본문 라벨 중 무엇인지 애매하다.

Required change:

- 원고 안에서는 다음 중 하나로 통일한다.
  - 짧은 표 제목은 `#### 표. ...`
  - 본문 흐름에 필요한 해석은 표 앞 문장으로 이동
  - 출처/주의사항은 표 뒤 별도 문단
- 블로그에서는 표 제목을 `h4` 또는 `figcaption`으로 바꾼다.
- PPT에서는 `[표]` 라벨을 그대로 옮기지 않고 슬라이드 제목을 주장문으로 바꾼다.

## Final Note

이번 검수의 결론은 “내용 부족”이 아니다. 내용은 충분하다. 문제는 일부 구간에서 좋은 내용이 표와 긴 설명에 묻혀, 독자가 바로 붙잡아야 할 메시지가 늦게 보인다는 점이다.

책의 원고는 최종 독서 경험이면서 동시에 블로그, 강의안, PPT, 워크시트의 원천 자료다. 따라서 앞으로의 수정 기준은 문장 교정만이 아니라 **전달 구조 교정**이어야 한다. 독자가 한 화면에서 무엇을 이해하고, 무엇을 믿고, 무엇을 실행해야 하는지 구분되는 배치가 필요하다.
