# Additionals Rules

`Additionals/` 아래의 모든 콘텐츠는 반드시 이 책 전체 내용에 기반해야 한다.

이 디렉터리의 자료는 독립적인 별도 세계관이나 별도 커리큘럼을 만드는 용도가 아니다. 목적은 오직 다음 두 가지 중 하나여야 한다.
- 책의 핵심 내용을 다른 형식으로 강조한다.
- 책의 내용을 독자가 더 정확히 이해하도록 돕는다.

책의 범위를 넘어가는 내용은 허용 범위가 제한된다.
- 새로운 주장, 새로운 프레임, 새로운 방법론을 독립적으로 추가하는 용도로 사용하지 않는다.
- 책 밖의 정보를 넣더라도 책 내용을 보강하거나 설명하기 위한 경우에만 사용한다.
- 책의 서술과 충돌하는 확장 해석은 넣지 않는다.

출처 규칙:
- `Additionals/` 아래의 모든 문서에는 반드시 관련 출처를 링크로 포함한다.
- 가장 우선하는 출처는 `chapters/` 내부의 관련 장 문서다.
- 책 밖의 보조 자료를 인용했다면 그 링크도 함께 남기되, 책과의 연결 목적을 명시한다.
- 최종본뿐 아니라 초안에도 가능하면 출처 링크를 남긴다.
- 링크 경로는 현재 문서의 위치에 맞는 상대 경로를 사용한다.

권장 출처 표기 형식:

```md
## Sources
- Book: [Chapter 3](../chapters/03_당신의_통장은_죄가_없다_기억이_문제지.md)
- Book: [Chapter 8](../chapters/08_대충_합리적인_따뜻한_투자.md)
- External: [자료명](https://example.com) - 8장의 개념 설명을 보조하기 위한 참고 자료
```

도메인 구조:
- `financial_lectures/`: 재무 강의 자료
- `financial_coaching/`: 재무 코칭 자료
- `blog_posts/`: 블로그 포스팅 자료
- `content_marketing/`: 콘텐츠 마케팅 자료

공통 트리:
- `outlines/`: 개요, 커리큘럼, 목차, 구조 초안
- `drafts/`: 작성 중인 원고, 스크립트, 세션 노트
- `final/`: 확정본
- `assets/`: 이미지, 도표, 썸네일, 첨부 자산
- `references/`: 조사 메모, 참고 자료, 링크 정리

저장 규칙:
- 새 자료를 만들 때는 먼저 책 원고인지, 부가 콘텐츠인지 분류한 뒤 경로를 정한다.
- 재무 강의 자료는 `Additionals/financial_lectures/` 아래에만 저장한다.
- 재무 코칭 자료는 `Additionals/financial_coaching/` 아래에만 저장한다.
- 블로그 포스팅 자료는 `Additionals/blog_posts/` 아래에만 저장한다.
- 콘텐츠 마케팅 자료는 `Additionals/content_marketing/` 아래에만 저장한다.
- 하나의 작업은 가능한 한 하나의 도메인 폴더 안에서 닫히도록 저장하고, 공용 참고 자료만 `references/`에 둔다.
- 파일명은 ASCII 기반의 소문자 `snake_case`를 기본으로 사용한다.

작업 체크리스트:
- 이 문서가 책 전체 내용과 연결되는가
- 연결되는 `chapters/` 파일 링크를 포함했는가
- 책의 범위를 넘는 정보라면 보조 목적이 분명한가
- 독립적인 새 주장처럼 읽히지 않는가
